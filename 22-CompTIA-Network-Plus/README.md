## ✨ Why This Certification Matters

### 🎯 The promise

CompTIA Network+ (N10-009) is the credential that takes you from a help-desk technician who *uses* the network to an engineer who *understands* it, every OSI layer, every routing protocol, every subnet, every wireless standard, every troubleshooting tool. It is the most-cited networking credential in entry-level network and sysadmin job postings, the explicit prerequisite for the next layer of cybersecurity certifications, and the cleanest way to prove networking fluency without spending two years and $300 on the Cisco CCNA. Pass Network+ and you have an industry-recognized credential that opens the door to network administration, cybersecurity, and cloud-infrastructure roles.

### 💼 Career outcomes after passing

- **Junior Network Administrator** ($65K–$90K), owning a single site's LAN (Local Area Network) / WAN (Wide Area Network) / Wi-Fi
- **Network Technician / NOC Analyst** ($60K–$85K), Tier-1 / Tier-2 monitoring and triage
- **Systems Administrator (network-leaning)** ($70K–$100K), bridge between sysadmin and net-admin
- **Network Engineer (2–3 years experience)** ($85K–$120K), design and operate enterprise networks
- **Cybersecurity Analyst** ($75K–$110K), Network+ is widely listed as the floor; Security+ (course 09) is the natural next step
- **Wireless / Mobility Specialist** ($80K–$115K), 802.11ax/be design, site surveys, Wi-Fi 6E rollouts
- **Cloud Network Engineer** ($110K–$150K), Network+ plus AWS (Amazon Web Services) / Azure credentials

Network+ sits on the second rung of CompTIA's "Infrastructure" and "Cybersecurity" pathways: **A+ (course 21) → Network+ (this course) → Security+ (course 09) → Linux+ / Server+ / CySA+ / PenTest+ → CASP+**. CompTIA's salary data shows Network+ holders earning roughly **20–30% more** than A+-only peers.

## 🎯 What You'll Learn

By the end of this track, you'll be able to:

- Read any network diagram and recite which OSI layer every protocol/device lives at
- Subnet IPv4 on paper in under 60 seconds, including VLSM with hand-calculated host counts
- Pick the right routing protocol (RIP/OSPF (Open Shortest Path First)/EIGRP/BGP) for any scenario and justify the choice
- Configure and troubleshoot VLANs, 802.1Q trunks, STP, and link aggregation by name
- Design a wireless LAN, 2.4 vs 5 vs 6 GHz, channel reuse, WPA3, captive portals, site surveys
- Recognize every common DNS (Domain Name System) record type, DHCP (Dynamic Host Configuration Protocol) scope concept, and cloud-connectivity option
- Apply the CompTIA 7-step troubleshooting methodology to L1–L7 problems and explain *why* each step exists
- Use ping, traceroute, tracert, nslookup, dig, ipconfig/ifconfig/ip, netstat, Wireshark, and SNMP (Simple Network Management Protocol)/syslog/NetFlow without Googling syntax

## ⚠️ The 7 Most Common Reasons People Fail

1. ❌ **Skipped subnetting practice**, Subnetting shows up everywhere. Many candidates panic when asked to find the network ID of `172.16.150.20/22`. Drill until it's reflex.
2. ❌ **Memorized port numbers but ignored the protocol behavior**, Knowing 443 = HTTPS (HTTP Secure) isn't enough; the exam asks WHY TLS (Transport Layer Security) uses TCP and what record types DNS uses on UDP (User Datagram Protocol) 53 vs TCP 53.
3. ❌ **Skipped the PBQs in practice**, They're worth a lot of points. PBQs include subnetting calculators, drag-and-drop OSI layers, and simulated CLI (Command Line Interface)/router config. Practice them.
4. ❌ **Confused similar wireless standards**, 802.11ac (Wi-Fi 5, 5 GHz only), 802.11ax (Wi-Fi 6/6E, 2.4 + 5 + 6 GHz), 802.11be (Wi-Fi 7). Flashcards fix this.
5. ❌ **Ignored troubleshooting methodology**, Domain 5 is 24%. Knowing "identify → theory → test → plan → implement → verify → document" in order is worth easy points.
6. ❌ **No time management**, 90 questions in 90 minutes = 60 seconds each. Don't get stuck on PBQs early. Flag and move on.
7. ❌ **Studied old N10-008 material**, The N10-009 exam (June 2024) added cloud connectivity (SD-WAN, SASE), Wi-Fi 6E/7, IPv6 deeper coverage, and Zero Trust. Use 009-current resources only.
