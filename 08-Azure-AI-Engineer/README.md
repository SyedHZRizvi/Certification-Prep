## 🎯 What You'll Learn

By the end of this track, you'll be able to:

- Choose the right Foundry services and models (LLMs, small models, multimodal, Foundry Tools) for a given generative or agentic task
- Set up a Microsoft Foundry project, configure model and agent deployments, and wire it into CI/CD
- Build generative applications with the Foundry SDK, including RAG, tool-augmented flows, and multistep reasoning
- Design and orchestrate single agents and multi-agent solutions with memory, function-calling, and tool schemas
- Stand up retrieval and grounding pipelines with Azure AI Search (vector, hybrid, semantic) and Content Understanding
- Evaluate models and agents for fabrications, relevance, quality, and safety, and instrument them with tracing and token analytics
- Apply Microsoft's Responsible AI controls with Content Safety, prompt shields, groundedness checks, and agent oversight modes
- Read and write the Python SDK calls the exam loves to test

## ⚠️ The 7 Most Common Reasons People Fail AI-103

1. ❌ Studied for the old AI-102 prebuilt-services exam and never built an agent or a Foundry app end to end
2. ❌ Memorized service names but couldn't tell which SDK method or Foundry connector to call in a code snippet
3. ❌ Confused Azure AI Search **indexer** with **index** with **skillset** (3 different things) in a RAG ingestion flow
4. ❌ Couldn't distinguish a **single agent with tools** from an **orchestrated multi-agent solution**, or say when to use each
5. ❌ Skipped Responsible AI and evaluation because "it's just policy", instrumentation, evaluators, and agent oversight are woven through the two heaviest domains
6. ❌ Tried to memorize old terminology, Microsoft renamed half the services (Form Recognizer → Document Intelligence, Cognitive Services → Azure AI services, AI Studio → Azure AI Foundry / Microsoft Foundry)
7. ❌ Underestimated case-study questions, they look like 1 question but are actually 4–6 sub-questions
