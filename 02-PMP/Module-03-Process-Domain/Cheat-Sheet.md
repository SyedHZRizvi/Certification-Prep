## 📅 Schedule

### Critical Path
- **Longest** path through network
- Determines **shortest** project duration
- **Zero float** on critical tasks

### Compression
| Crashing | Fast-tracking |
|----------|---------------|
| Add resources | Run in parallel |
| 💰 More cost | 🆓 Often no cost |
| Limited risk | 🚨 More risk |

### Estimation
- **Analogous**, like past projects (fast)
- **Parametric**, math relationships
- **Three-point (PERT)**, `(O+4M+P)/6`
- **Bottom-up**, sum of details (slowest, most accurate)

### Estimate Ranges
| Type | Range |
|------|-------|
| ROM | -25% to +75% |
| Budgetary | -10% to +25% |
| Definitive | -5% to +10% |

### Reserves
| Type | For | Part of baseline? |
|------|-----|-------------------|
| **Contingency** | Known risks | ✅ Yes |
| **Management** | Unknown risks | ❌ No (sponsor) |

## 🔄 Change Control

1. Receive change request
2. Evaluate impact
3. CCB review
4. Approve/Reject
5. Update plan
6. Communicate

🎯 NEVER: skip analysis or just add changes.

---

## 🎯 Project Charter

- Signed by: **Sponsor**
- Gives PM authority
- High-level: purpose, objectives, milestones, risks, budget

---

## 🏁 Closure
1. Confirm scope done
2. Formal acceptance
3. Lessons learned (capture throughout!)
4. Archive
5. Release resources
