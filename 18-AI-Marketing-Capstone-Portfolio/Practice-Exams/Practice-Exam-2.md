### Key Grading Themes For This Exam

**The "Did You Ship?" Block (Q1, Q3, Q4, Q5, Q7, Q24, Q42)**: These questions test whether work is *live and public*, not just drafted. The single biggest failure mode in this course is "I built it but it's not public."

**The "Honest About Failure" Block (Q9, Q11, Q12, Q14, Q18, Q27, Q36, Q53)**: Hiring managers and clients trust portfolios that show what didn't work. Sanitized portfolios read as fake.

**The "Audience-Ready" Block (Q16, Q23, Q33, Q39, Q44, Q49, Q50)**: This block tests whether your portfolio is positioned for the audience you said you'd target. If you're job-tracking but everything is freelance-flavored, fix that.

**The "Coherence" Block (Q34, Q40, Q51, Q52)**: Are the 5 capstones one marketer's voice, or 5 random pieces? Coherence multiplies the effect.

**The "Mindset" Block (Q31, Q57, Q58, Q59, Q60)**: The course only "works" if you keep going. These questions test whether you've internalized that.

## Detailed answer rationales

> Practice Exam 2 covers the full 5-capstone + LinkedIn + portfolio + outreach picture. Every question maps to a specific module's "did you ship at the bar" check. The rationales below explain the highest-impact questions, why "A" is the Excellent benchmark and what each lower option tells you about the gap.

**Q1. Answer: A "Yes all 5 done + published"**

**Why A is the Excellent benchmark.** The course is structurally built around 5 capstones, each chosen for a specific hiring signal (SEO = analytical writing, paid = money-at-stake, automation = systems, attribution = quantitative, AI agent = modern leverage). All 5 published gives you full coverage; the wrapper (LinkedIn + portfolio + pitch) works on top of complete coverage.

**Why the other options fall short.**
- **B (4 of 5 done)**: You're missing one hiring signal. If you skipped the AI agent for a B2B SaaS role, you've cut the most-differentiated piece. If you skipped paid campaign, you've cut the only money-at-stake credibility signal.
- **C (3 of 5 done)**: The portfolio reads as "started a course, didn't finish." Worse than 5 different mediocre pieces.
- **D (<3 done)**: The portfolio doesn't function. Re-do the missing capstones before continuing to Module 7.

**Exam-takeaway.** Each capstone is load-bearing for a specific hiring signal. Skipping one isn't a 20% loss, it's a structural gap.

---

**Q7. Answer: A "Yes 5 sent"**

**Why A is the Excellent benchmark.** Module 8 is operationally "send 5 messages this week." Without the sending, every other deliverable in the course (deck, proposal template, outreach scripts) is theatre. The whole portfolio exists to convert into conversations, sending IS the test.

**Why the other options fall short.**
- **B (3–4 sent)**: Stopped one short of the calibrated batch size. Usually fear-driven. The fix is just to send the last 1–2.
- **C (1–2 sent)**: You did the easy ones and stopped at the hard ones. This is the most-common course-completion failure.
- **D (None sent)**: The course didn't conclude. The work doesn't matter without distribution.

**Exam-takeaway.** Hiring outcomes are bottlenecked by message-sending, not capstone-building. If you've built but not sent, you're 95% done with 0% of the result.

---

**Q13. Answer: A "Yes public + README solid"**

**Why A is the Excellent benchmark.** A public GitHub repo with a solid README is the "GitHub equivalent" for the AI agent capstone, the artifact a hiring manager can verify in 30 seconds. The README is the front door; most recruiters never run your code.

**Why the other options fall short.**
- **B (Public, README thin)**: The repo exists but the README doesn't explain what it does. Recruiter sees a wall of code, bounces.
- **C (Private repo)**: Recruiter can't see anything. Functionally the same as no portfolio piece.
- **D (Not on GitHub)**: No verifiable artifact exists. The agent doesn't function as a hiring signal.

**Exam-takeaway.** README quality > code quality for portfolio purposes. A clean README on simple code beats a brilliant agent buried behind a missing README.

---

**Q15. Answer: A "Yes verified"**

**Why A is the Excellent benchmark.** Verifying .env is not in the public repo is the single most-embarrassing portfolio failure to avoid. Leaked API keys = immediate trust failure + actual financial exposure.

**Why the other options fall short.**
- **B (Excluded but unverified)**: You added .env to .gitignore but didn't check if it was committed BEFORE the gitignore was added. Common error.
- **C (Not checked)**: Probably fine, but you don't know, and recruiters who run `git log --all -- .env` will know.
- **D (Keys committed)**: ROTATE NOW. Then use `git-filter-branch` or BFG Repo-Cleaner to scrub history. Then re-push.

**Exam-takeaway.** Five-minute verification (`git log --all -- .env` and `git log --all --source -p | grep -i "api[-_]key"`) beats catastrophic key leak. Do it before publishing.

---

**Q18. Answer: A "Yes 2+ testimonials"**

**Why A is the Excellent benchmark.** Testimonials from capstone subjects (1-line quotes from the indie founder, friend, or business whose data you audited) are higher-trust signals than any self-claim you can make. 2+ means you've built a verifiable track record across multiple subjects.

**Why the other options fall short.**
- **B (1 testimonial)**: Better than zero. Single testimonial can be perceived as cherry-picked.
- **C (Asked but no reply)**: Ask again. Follow up. Most subjects forget; a nudge works.
- **D (Haven't asked)**: Easy fix, send the ask emails today. Most subjects will say yes.

**Exam-takeaway.** Ask every capstone subject for a 2-line testimonial. Conversion rate ~50%. Each one is worth more than 5 paragraphs of self-promotion.

---

**Q31. Answer: A "Yes that's the math, calm with it"**

**Why A is the Excellent benchmark.** Cold-outreach math is empirically: 5 messages → ~1 reply → ~1 conversation → ~30% close rate. Being mentally calm with 80% ghost rate is the precondition for sending the *next* batch of 5. Without that mental calibration, students burn out after one batch.

**Why the other options fall short.**
- **B (Mostly OK)**: You'll send the next batch but with reduced volume.
- **C (Hopes are high)**: You'll be deflated when 4/5 ghost and stop the practice.
- **D (Anxious about it)**: You'll send 5, see no replies in 3 days, and quit the outreach machine entirely.

**Exam-takeaway.** Outreach success is a function of *time*, not *batch quality*. The math compounds across batches, survive 4 weeks at 5 messages/week and you'll have ~3 conversations in flight.

---

**Q35. Answer: A "Yes within range, live exec"**

**Why A is the Excellent benchmark.** A 3-5 min Loom with live execution is the single highest-leverage content piece in the AI Agent capstone. Recruiters who'd never read 500 lines of code will watch a 4-minute Loom of the agent doing its job.

**Why the other options fall short.**
- **B (Recorded, slightly off)**: Minor off-time (2 min or 7 min) is fine. The risk is just less engagement.
- **C (Code review only)**: Without live execution, the agent "doesn't run" in the recruiter's mental model. They can't tell if it actually works.
- **D (No Loom)**: Recruiters won't run your code. Without the Loom, the agent is invisible.

**Exam-takeaway.** Live execution in the Loom is the trust-builder. Anyone can paste code; only people whose agent actually runs can demo it executing on camera.

---

**Q52. Answer: A "Yes feels like one marketer's voice"**

**Why A is the Excellent benchmark.** Coherence multiplies the effect. 5 case studies that read like the same person wrote them feel like a body of work. 5 disjointed pieces even if each is individually solid feel like a junior who's tried lots of things.

**Why the other options fall short.**
- **B (Mostly)**: One outlier piece breaks the body. Usually the AI agent or attribution model, where students slip into technical jargon.
- **C (Mixed)**: Reading the portfolio feels like clicking between 5 different blogs.
- **D (Disjointed)**: Hiring manager can't form a coherent mental model of who you are. Result: nothing memorable.

**Exam-takeaway.** Voice consistency is a craft skill. Read all 5 case studies aloud in one sitting, if they don't sound like the same writer, rewrite the outliers.
