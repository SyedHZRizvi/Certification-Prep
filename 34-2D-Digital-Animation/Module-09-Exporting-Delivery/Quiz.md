## 🎯 Answers + Explanations

```
Q1.  B, H.264 (MP4) is the most universally compatible codec for web/social delivery.
Q2.  C, H.264 requires Adobe Media Encoder; it is not available directly from AE Render Queue.
Q3.  C, WebM (VP9) supports alpha channels natively in web browsers.
Q4.  B, Lottie is based on Shape Layer (vector path) data from After Effects.
Q5.  C, GIF's 256-color limit causes visible banding in gradients and photographic content.
Q6.  B, ProRes 4444 is chosen when alpha (transparency) channel is required in the output.
Q7.  B, YouTube standard: H.264 MP4 at 1920×1080 (or higher) at native frame rate.
Q8.  B, Bodymovin exports AE compositions to Lottie JSON format.
Q9.  B, sRGB and Rec.709 share primaries but have different gamma curves; mismatch = color shift.
Q10. B, TikTok: H.264 MP4, 1080×1920 portrait (9:16), 30fps.
Q11. A, 24 and 30 don't share a common factor; frame blending/duplication degrades quality.
Q12. D, Lottie JSON is vector-based and designed for iOS/Android app embedding.
Q13. B, US broadcast TV: ProRes 422 at 29.97fps NTSC standard.
Q14. B, Color space mismatch (sRGB working space vs. Rec.709 delivery) causes washed-out look.
Q15. B, Batch processing = multiple export jobs queued and rendered sequentially/in parallel.
Q16. B, Unsupported AE features (3D, raster footage, certain effects) don't export in Lottie.
Q17. C, European broadcast = 25fps PAL standard.
Q18. B, WebM: smaller files, better quality, alpha support vs. GIF's color limitations.
Q19. C, WebM with alpha (VP9) is the correct choice for transparent web animation.
Q20. B, Composition → Add to Adobe Media Encoder Queue → set H.264 preset → Start Queue.
Q21. C, Rec.2020 is the HDR TV standard for very wide gamut (HDR10, Dolby Vision).
Q22. B, PNG sequence: lossless, per-frame alpha, ideal for compositing pipelines.
Q23. B, File → Project Settings → Color → Working Color Space controls AE color management.
Q24. B, ProRes 4444 provides lossless alpha and archival quality for future compositing use.
```
