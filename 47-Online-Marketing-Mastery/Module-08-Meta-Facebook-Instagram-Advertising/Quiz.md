## Answer Key

| Q | Ans | Explanation |
|---|-----|-------------|
| 1 | B | The correct hierarchy is Campaign (objective) → Ad Set (audience/budget/placement) → Ad (creative). "Ad Group" is Google's terminology, not Meta's. |
| 2 | C | The Sales objective tells Meta to optimize delivery for users likely to complete a purchase event on your website. Traffic optimizes for clicks only. |
| 3 | B | CBO (now called Advantage Campaign Budget) automatically distributes the campaign-level budget across ad sets to maximize overall campaign results. |
| 4 | B | The Pixel fires JavaScript events from the user's browser and sends them back to Meta, enabling remarketing, conversion tracking, and attribution. |
| 5 | C | "ClickThrough" is not a standard Meta event. Standard events include Purchase, AddToCart, ViewContent, Lead, CompleteRegistration, InitiateCheckout, and Search. |
| 6 | C | iOS 14.5 introduced ATT — users who opted out of tracking could no longer be tracked via the browser-based Pixel, causing reported conversions to drop sharply even if actual sales continued. |
| 7 | B | CAPI sends event data from the advertiser's server (not the user's browser) directly to Meta, bypassing ATT and browser privacy restrictions. |
| 8 | C | Meta uses the `event_id` parameter (sent by both the Pixel and CAPI) to identify and deduplicate events, preventing double-counting. Advertisers must ensure unique event_ids are generated. |
| 9 | B | Under UK GDPR (retained post-Brexit) and EU GDPR, uploading customer data to Meta as a Custom Audience requires valid legal basis (consent or legitimate interest, typically consent for marketing) and disclosure in the privacy policy. |
| 10 | B | AEM is Meta's privacy framework introduced after iOS 14.5, limiting advertisers to 8 prioritized conversion events per verified domain for attribution on Apple devices. |
| 11 | B | Purchase is highest priority (the most valuable event), followed by AddToCart, InitiateCheckout, ViewContent. Under AEM, if a user triggers Purchase, only Purchase is credited — the hierarchy ensures the most valuable signal is always captured. |
| 12 | B | Meta requires a minimum of 100 people in a seed audience to create a Lookalike, but recommends 1,000–50,000 for quality results. |
| 13 | B | A 1% Lookalike in the UK finds the 1% of UK Meta users most similar to the seed audience — typically several hundred thousand people, not a copy of the seed list. |
| 14 | B | Advantage+ Audience allows Meta's algorithm to expand targeting beyond the specified parameters when it predicts better conversion outcomes — giving Meta more freedom to find the right users. |
| 15 | B | ASC combines prospecting (finding new customers) and retargeting (reaching people who have previously interacted) into one campaign, with Meta controlling the budget split via AI. |
| 16 | C | A frequency of 4.2 for a cold audience indicates creative fatigue. The correct response is to rotate creatives, expand the audience, or introduce fresh ad sets. Pausing 30 days is extreme and unnecessary. |
| 17 | C | Meta recommends 50 conversions per ad set per week to exit the learning phase. Below this threshold, the algorithm cannot gather enough data to optimize efficiently. |
| 18 | C | The view occurred on Monday. The 1-day view window expired on Tuesday. There was no click, so there is no 7-day click window open. The purchase on the following Sunday falls outside both windows and would not be attributed. |
| 19 | B | Carousel ads allow 2–10 cards, each with its own image, headline, description, and URL — ideal for showcasing a multi-step tutorial or multiple product features sequentially. |
| 20 | B | CAPI involves transmitting personal data (including hashed PII) from the advertiser's server to Meta. This constitutes data processing under GDPR, requiring legal basis (typically consent), a DPA with Meta, and proper disclosure. |
| 21 | B | EMQ is a score from 0–10 that reflects how accurately CAPI event data can be matched to a Meta user profile. Higher EMQ means better attribution. Passing more customer parameters (hashed email, phone, name, city) improves EMQ. |
| 22 | B | A valid A/B test changes only ONE variable (creative in this case) and uses Meta's Experiment tool which enforces mutual audience exclusion — ensuring the same user cannot see both variants, which would contaminate results. |
| 23 | A | After CAPI is implemented, Meta will receive conversion signals it previously couldn't see (from users whose browsers blocked the Pixel). This typically increases reported conversions and therefore reported ROAS. Deduplication via event_id prevents double-counting. |
| 24 | B | Dynamic Ads automatically pull from a Meta product catalogue to show personalized product recommendations — most commonly used to retarget users who viewed specific products on your website without purchasing. |
| 25 | B | The gold standard measurement stack combines Pixel + CAPI (deduplicated), domain verification, AEM with 8 prioritized events, a compliant consent management platform (IAB TCF for EU), and a 7-day click / 1-day view attribution window aligned to Meta's current model. |
