## 👗 A Story: How Stitch Fix Built a $1B Predictive-Marketing Engine

Recall the Stitch Fix story from Module 2, the activation threshold was the **3rd Fix**. Customers who reached three Fixes had 85% twelve-month retention; customers who churned after Fix #1 had 11% retention. The strategic question for the marketing-science team was: *for every customer in the first 30 days, what is the probability they will reach Fix #3?*

The team built a stacked ensemble model:

- **Feature input**: 142 features from the 60-question style-quiz, the first-Fix item selections, the keep/return decisions, browse behavior on the site, demographic data, geographic data, and stylist-note sentiment scores.
- **Model**: Gradient-boosted trees (LightGBM) → output probability of reaching Fix #3.
- **Calibration**: Platt scaling so the predicted probability matched the empirical rate.
- **Output AUC**: 0.84 on a held-out test set, strong, not perfect, but operationally useful.

The marketing-science team then plugged this model into three downstream systems:

1. **Audience scoring**: every customer was scored daily and segmented into deciles by "probability to reach Fix #3."
2. **Ad bidding**: the top 3 deciles were synced as a Meta Custom Audience for bid-up; bottom 3 deciles synced as a "do not target, likely churners" audience for bid-down.
3. **Lifecycle messaging**: middle deciles got automated nudge campaigns ("Try one more Fix, here's $20 off") with copy generated by a templating system that mentioned the customer's style preferences from the first Fix.

The composite ROI of the predictive-scoring program, measured by a quarterly geo-holdout test: **24% lift in 90-day cohort revenue** versus a control region using non-personalized bidding and no nudges. At Stitch Fix's revenue scale, that lift translates to roughly $290M in incremental annual revenue.

This is the canonical case study of *what predictive analytics for marketing actually does*. This module teaches you to build the same thing.

## 💎 Customer Lifetime Value (CLV), Why It Matters

Customer Lifetime Value is the total revenue (or profit) a single customer is expected to generate over the lifetime of their relationship with your business. The strategist's job is to estimate it *for each customer individually*, so that:

- **Marketing investment** can be sized by customer value (don't pay $200 CAC for a $80 LTV customer).
- **Customer service** can be triaged by value (highest-LTV customers go to senior reps).
- **Product development** can be prioritized by what high-LTV customers want.
- **Channel mix** can be optimized for *LTV-acquiring* channels, not *cheap-CAC* ones.

### A simple (and wrong) CLV formula

The most-quoted CLV formula, beloved by MBA textbooks:

```
CLV = (Average Order Value) × (Purchase Frequency) × (Customer Lifespan) × Gross Margin
```

This works for *steady-state, contractual* businesses (Netflix, Spotify) where you can directly observe lifespan. For *non-contractual* businesses (most e-commerce, most consumer apps) it's wrong, because:

- You can't directly observe lifespan (no churn signal).
- AOV and frequency vary heavily by customer.
- The formula gives one number per business, not per customer.

### A better model, BG/NBD + Gamma-Gamma (Fader & Hardie)

In 2005, Peter Fader (Wharton) and Bruce Hardie (LBS) published a series of papers establishing the **Beta-Geometric / Negative Binomial Distribution (BG/NBD)** model for predicting the number of future purchases by a non-contractual customer. Combined with the **Gamma-Gamma model** for predicting average order value per customer, this gives a probabilistic, per-customer CLV.

#### BG/NBD model assumptions

1. While **active**, a customer makes purchases according to a **Poisson process** with rate `λ`. Different customers have different `λ`s drawn from a **Gamma distribution** with shape `r` and scale `α`.
2. After every purchase, the customer "drops out" (becomes inactive forever) with probability `p`. Different customers have different `p`s drawn from a **Beta distribution** with parameters `a` and `b`.
3. Customers' purchase rates and dropout probabilities are **independent**.

The model takes four inputs **per customer**:

- `x` = number of repeat transactions (i.e., total purchases minus the first)
- `t_x` = time of the most recent transaction
- `T` = time of observation (now)
- `monetary_value` = average transaction value

And outputs:

- **Expected number of future transactions** in the next `t` time units.
- **Probability the customer is still active**.
- (With Gamma-Gamma) **Expected revenue per future transaction**.

Multiply the two to get CLV.

#### The math (you should at least recognize this)

The BG/NBD likelihood for a single customer:

```
P(X(T)=x, t_x, T | r, α, a, b)
   = B(a, b+x) / B(a, b) × Γ(r+x) / Γ(r) × α^r / (α + T)^(r+x)
   + δ_{x>0} × B(a+1, b+x-1) / B(a, b) × Γ(r+x) / Γ(r) × α^r / (α + t_x)^(r+x)
```

Where `B(·,·)` is the Beta function and `Γ(·)` is the Gamma function. In practice, you don't compute this yourself, you call **`lifetimes.fit()`** in Python.

### Worked Python example using the `lifetimes` library

```python
import pandas as pd
from lifetimes import BetaGeoFitter, GammaGammaFitter
from lifetimes.utils import summary_data_from_transaction_data

# Load transaction history
transactions = pd.read_csv('transactions.csv')
# Required columns: customer_id, transaction_date, monetary_value

# Summarize into BG/NBD-ready format
summary = summary_data_from_transaction_data(
    transactions,
    customer_id_col='customer_id',
    datetime_col='transaction_date',
    monetary_value_col='monetary_value',
    observation_period_end='2026-05-01'
)
# summary now has: frequency (= x), recency (= t_x), T, monetary_value

# Fit BG/NBD
bgf = BetaGeoFitter(penalizer_coef=0.001)
bgf.fit(summary['frequency'], summary['recency'], summary['T'])

# Predict expected transactions in next 90 days
summary['predicted_purchases_90d'] = bgf.conditional_expected_number_of_purchases_up_to_time(
    90,                       # forecast horizon in days
    summary['frequency'],
    summary['recency'],
    summary['T']
)

# Predict probability of being alive
summary['probability_alive'] = bgf.conditional_probability_alive(
    summary['frequency'], summary['recency'], summary['T']
)

# Fit Gamma-Gamma for monetary value
# Note: Gamma-Gamma requires no correlation between frequency and monetary_value
returning = summary[summary['frequency'] > 0]
ggf = GammaGammaFitter(penalizer_coef=0.01)
ggf.fit(returning['frequency'], returning['monetary_value'])

# Compute CLV (gross, before margin)
summary['predicted_clv_12mo'] = ggf.customer_lifetime_value(
    bgf,
    summary['frequency'],
    summary['recency'],
    summary['T'],
    summary['monetary_value'],
    time=12,                   # 12 months
    freq='D',                  # daily intervals in input
    discount_rate=0.01         # monthly discount for NPV
)

print(summary.head())
```

The output: every customer has a **predicted CLV over the next 12 months** with a probabilistically sound foundation. Sort by predicted CLV and you have your customer scoring for ad targeting, service triage, and retention investment.

## 🚪 Churn Prediction

Where BG/NBD is right for *non-contractual* businesses, **classification models** are right for *contractual* businesses (subscriptions). The canonical churn question: *what is the probability this customer churns in the next 30 days?*

### Feature engineering for churn

| Feature category | Examples |
|------------------|----------|
| Recency | Days since last login / purchase / engagement |
| Frequency | Sessions per week, weeks active in last 90 days |
| Monetary | Spend in last 30 / 60 / 90 days; trajectory |
| Engagement | Feature usage, content consumed, support tickets |
| Customer support | Number of tickets, sentiment of last interaction |
| Tenure | Days since signup |
| Plan changes | Upgrades, downgrades, pauses |
| Demographic | Age, geography, life stage |
| Cohort | Acquisition channel, acquisition month |

### Model choice

| Model | When to use |
|-------|-------------|
| Logistic regression | Baseline; interpretable coefficients; quick to ship |
| Gradient-boosted trees (XGBoost, LightGBM, CatBoost) | Default for most production churn models; handles non-linearity well |
| Random forest | Simpler than GBT; rarely beats LightGBM |
| Neural network | When you have >1M customers + rich behavioral sequences |
| Survival analysis (Cox PH, accelerated failure time) | When you want time-to-event prediction, not just classification |

The vast majority of production churn models in 2026 are **LightGBM** or **CatBoost**, they handle missing data, mixed types, and class imbalance well, train fast, and are interpretable via SHAP.

### Worked Python example, churn classification with LightGBM

```python
import lightgbm as lgb
import pandas as pd
from sklearn.model_selection import train_test_split
from sklearn.metrics import roc_auc_score, classification_report
from sklearn.calibration import CalibratedClassifierCV

# Load feature matrix
df = pd.read_csv('churn_features.csv')
# Columns: customer_id, [40+ features], churned_30d (target: 1 if churned in 30d)

X = df.drop(columns=['customer_id', 'churned_30d'])
y = df['churned_30d']

# Train/test split (stratified to preserve class balance)
X_tr, X_te, y_tr, y_te = train_test_split(
    X, y, test_size=0.2, stratify=y, random_state=42
)

# LightGBM with hyperparams tuned for class imbalance (e.g., 5% churn rate)
model = lgb.LGBMClassifier(
    n_estimators=2000,
    learning_rate=0.03,
    max_depth=6,
    num_leaves=31,
    class_weight={0: 1, 1: 18},        # weight positive class heavier
    reg_alpha=0.1,
    reg_lambda=0.1,
    random_state=42
)

# Calibrate (Platt scaling) so output probabilities are meaningful
calibrated = CalibratedClassifierCV(model, method='isotonic', cv=5)
calibrated.fit(X_tr, y_tr)

# Evaluate
proba = calibrated.predict_proba(X_te)[:, 1]
print(f"Test AUC: {roc_auc_score(y_te, proba):.3f}")

# Predict on production
preds_production = calibrated.predict_proba(X_production)[:, 1]
```

### Evaluating model quality

| Metric | What it measures | Good value |
|--------|------------------|------------|
| **AUC-ROC** | Discrimination across thresholds | >0.80 for marketing churn (>0.85 great) |
| **Precision @ top decile** | Hit rate of your highest-risk segment | >3× base rate |
| **Lift @ top decile** | Times more likely to churn than random | >3× |
| **Brier score** | Calibration of probabilities | Lower is better; <0.05 is great |
| **Calibration plot** | Predicted vs actual probability | Should hug the y=x line |
| **AUC-PR** | Better than AUC-ROC for imbalanced data | Depends on base rate |

🎯 **Strategist's bar:** A churn model is *operationally useful* if lift in the top decile is ≥3×. Below 2×, the model is barely better than random; above 5×, you may be overfitting.

### Why calibration matters

A LightGBM model trained naively often outputs *uncalibrated* probabilities, it might output 0.6 for a customer whose *empirical* churn rate is 0.2. For marketing decisions ("retain customers above 30% churn probability"), you need the output to actually mean "30% of customers with this score will churn." Use **isotonic regression** or **Platt scaling** to calibrate.

## 🎯 Propensity-to-Buy Scoring

Propensity scoring is the *positive* mirror of churn, predicting the probability a customer will *do* something desirable (buy, upgrade, refer, attend an event) in a defined window.

Mechanically, propensity scoring uses the same model classes as churn (LightGBM, logistic regression, etc.). The strategic difference is in **the action you take** on the score:

| Decile | Action |
|--------|--------|
| 1 (top 10% propensity) | Bid-up in paid social, lookalike-seed audience, premium email |
| 2–3 | Targeted nudge with offer |
| 4–7 | Brand-only contact, no offers |
| 8–10 (low propensity) | Suppress from paid acquisition entirely |

⚠️ **What most teams get wrong:** They retarget the top decile (highest propensity) in paid ads. *These users are the most likely to convert without ads.* This destroys ad ROAS. The right move: target the middle deciles, where the model predicts the action is *plausible but not certain*, i.e., where marketing has the most incremental value.

This is where **uplift modeling** comes in.

## 🚀 Uplift Modeling (the strategist's secret weapon)

A **propensity** model predicts *who will buy*. An **uplift** model predicts *who will buy because of the marketing intervention*. These are different.

Customers segment into four uplift categories:

| Type | Without marketing | With marketing | Should you market? |
|------|-------------------|----------------|---------------------|
| **Persuadables** | No buy | Buy | YES (the incremental wins) |
| **Sure things** | Buy | Buy | NO (wasted impressions) |
| **Lost causes** | No buy | No buy | NO |
| **Do-not-disturbs / Sleeping dogs** | Buy | No buy | NO (marketing actively backfires) |

Uplift modeling tries to identify the **Persuadables** specifically. It requires a **randomized treatment / control** dataset, typically generated by deliberately not marketing to a holdout group.

### Two common uplift approaches

1. **Two-model approach (T-learner)**: train one model on the treatment group and one on the control group. Predict each customer with both. Uplift = treatment-prediction − control-prediction.
2. **Single-model approach**: train a model with the treatment indicator as a feature. Compute predictions with treatment = 1 and treatment = 0. Uplift = difference.

The Python library **`causalml`** (by Uber) and **`scikit-uplift`** are the canonical open-source uplift tools.

### A worked example, uplift modeling in causalml

```python
from causalml.inference.tree import UpliftTreeClassifier

# Data: customer features X, treatment (1=marketed, 0=control), outcome (1=converted)
upl = UpliftTreeClassifier(max_depth=5, min_samples_leaf=200, evaluationFunction='KL')
upl.fit(X.values, treatment=treatment.values, y=outcome.values)

# Score new customers
uplift_predictions = upl.predict(X_new.values)
# Returns probability uplift per customer

# Action: market only to customers with uplift > 0
target_audience = X_new[uplift_predictions > 0]
```

🎯 **Strategist's payoff:** Uplift models routinely achieve **2–3× the ROI** of propensity-only targeting in lifecycle campaigns because they avoid wasting impressions on Sure Things and Lost Causes.

## 📐 Segmentation via Clustering, RFM and Beyond

A *propensity model* gives you a number per customer. A *clustering model* gives you a group label.

### RFM segmentation (the classic)

**RFM** = Recency, Frequency, Monetary. Score each customer on each dimension (typically quintiles, 1–5), then group:

```
Recency   = 5 if last purchase within 7 days, 1 if >180 days
Frequency = 5 if >12 purchases/year, 1 if 1 purchase total
Monetary  = 5 if average order >$200, 1 if <$25

RFM score = concatenation, e.g., "555" = champion, "111" = lost
```

Canonical RFM segments:

| Segment | RFM scores | Strategy |
|---------|-----------|----------|
| Champions | 4-5, 4-5, 4-5 | Reward, refer, advocate |
| Loyal customers | 3-5, 4-5, 2-5 | Cross-sell, premium offers |
| Recent customers | 4-5, 1-2, 1-3 | Onboard, drive repeat |
| At-risk | 1-3, 4-5, 4-5 | Win-back campaigns |
| Lost | 1-2, 1-2, 1-2 | Reactivation or sunset |

### ML-extended RFM (the modern version)

In 2026, RFM is usually a *baseline feature input* to a clustering algorithm typically **K-means** or **HDBSCAN** that also incorporates behavioral, engagement, and demographic features. The output: clusters that you label semantically based on profile (e.g., "premium weekend buyers", "discount-driven mobile shoppers", "B2B procurement").

### Worked Python example, K-means clustering

```python
import pandas as pd
from sklearn.preprocessing import StandardScaler
from sklearn.cluster import KMeans

# Features
features = ['recency_days','frequency_90d','monetary_90d',
            'sessions_per_week','category_diversity',
            'discount_sensitivity','mobile_share']
X = df[features]

# Scale features
scaler = StandardScaler()
X_scaled = scaler.fit_transform(X)

# Find optimal K via elbow method (inertia + silhouette)
from sklearn.metrics import silhouette_score
for k in range(2, 10):
    km = KMeans(n_clusters=k, random_state=42, n_init=20)
    labels = km.fit_predict(X_scaled)
    sil = silhouette_score(X_scaled, labels)
    print(f"K={k}: silhouette={sil:.3f}, inertia={km.inertia_:.0f}")

# Fit with chosen K
km = KMeans(n_clusters=5, random_state=42, n_init=20)
df['cluster'] = km.fit_predict(X_scaled)

# Profile clusters
print(df.groupby('cluster')[features].mean())
```

The strategist then *labels* the clusters semantically and routes them to different lifecycle programs.

Not every marketing team has data scientists. The 2026 no-code stack:

| Tool | What it does | Strategist use case |
|------|--------------|---------------------|
| **DataRobot** | Automated ML, feed it a CSV + target, it builds models | Churn, propensity, CLV prediction with click-and-train |

The strategist's job is to *verify* the model before activating it, check AUC, calibration, lift, and SHAP feature importance.

## 🔢 A Worked Numerical Example, End-to-End

Suppose you run a DTC brand with 200,000 active customers. You:

1. **Build a 12-month CLV model** using BG/NBD + Gamma-Gamma. Output: each customer has a predicted CLV.
2. **Distribution check**, top decile predicted CLV: $620. Bottom decile: $35. Average: $135.
3. **Set CAC budget**, you decide you'll spend up to 30% of predicted 12-month CLV to acquire a customer. So top-decile-lookalike CAC ceiling = $186; bottom-decile-lookalike CAC ceiling = $10.
4. **Activate**, sync the top decile to Meta as a Custom Audience seed for Advantage+ Lookalikes, with a $186 ceiling bid. Sync the bottom decile as a "do not target" exclusion.
5. **Build a 30-day churn model** for existing customers. Top decile probability-to-churn: 0.38. Bottom decile: 0.02.
6. **Set retention budget**, you spend up to 20% of predicted remaining CLV on retention for customers with churn probability > 0.20. For a customer with $400 remaining predicted CLV and a 0.38 churn probability, you can spend up to $80 on a retention offer.
7. **Measure** with a quarterly geo-holdout to validate the *incremental* impact of both programs.

This is the strategist's playbook. Memorize this six-step flow.

## ⚠️ Common Misconceptions

| Misconception | Reality |
|---------------|---------|
| "CLV is a single number for the business" | Modern CLV is per-customer, probabilistic |
| "The simple CLV formula works for everyone" | Only for contractual / steady-state businesses |
| "AUC is the only metric that matters" | Calibration + lift @ top decile matter more for marketing actions |
| "More features = better model" | Often the opposite, feature selection + regularization wins |
| "Propensity = uplift" | No, uplift requires randomized treatment data |
| "RFM is obsolete" | RFM is a *baseline*; modern clustering builds on RFM |
| "AutoML removes the need for ML knowledge" | AutoML *amplifies* the consequences of bad input data |
| "Higher predicted CLV always means market harder to them" | Top deciles often need *less* marketing, they're sure things |

## 🎓 Key Terms To Know

| Term | Definition |
|------|------------|
| **CLV / LTV** | Customer Lifetime Value, predicted total revenue from a customer |
| **CAC** | Customer Acquisition Cost |
| **LTV:CAC ratio** | Health metric; >3:1 typical bar for sustainable acquisition |
| **BG/NBD** | Beta-Geometric / Negative Binomial Distribution model for non-contractual CLV (Fader & Hardie 2005) |
| **Gamma-Gamma** | Companion model to BG/NBD for predicting monetary value per transaction |
| **Lifetimes library** | The canonical Python implementation of BG/NBD + Gamma-Gamma |
| **Churn model** | Classification model predicting whether a customer will leave in a window |
| **Propensity model** | Classification model predicting whether a customer will take a positive action |
| **Uplift model** | Predicts the *incremental* effect of marketing, Persuadables only |
| **Persuadables / Sure Things / Lost Causes / Do-Not-Disturbs** | The four uplift segments |
| **AUC-ROC** | Area under the ROC curve; discrimination metric |
| **Brier score** | Calibration metric (lower better) |
| **Calibration / Platt scaling / Isotonic regression** | Techniques to make output probabilities meaningful |
| **Lift @ top decile** | Operationally useful threshold ≥3× base rate |
| **RFM** | Recency, Frequency, Monetary, classic 3-dimensional customer scoring |
| **K-means / HDBSCAN** | Common clustering algorithms |
| **DataRobot / Vertex AI / Amazon Personalize** | No-code AutoML / managed-ML tools |
| **SHAP** | Feature importance via Shapley values for ML models |
