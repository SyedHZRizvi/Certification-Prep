## Detailed answer rationales

> Use these to learn the *discrimination* logic — why the right answer is right AND why each distractor is wrong. This is the case-method discipline that distinguishes Harvard Law-style learning from rote memorization.

**Q1. Answer: A — Inventory carrying cost.**
*Why A is correct.* Adding DCs multiplies safety stock per the square-root rule (√N rule) — inventory carrying is the cost that rises most steeply in proportion to network nodes.
*Why others are wrong.* B (Outbound transport) actually FALLS with more DCs (each DC closer to its customers); C (Service-level penalty) IMPROVES with more DCs; D (Stockout cost) IMPROVES too.
*Takeaway.* Trade-off curve: outbound transport down, inventory + facility cost up. The "rising" cost on the exam = inventory.

**Q2. Answer: C — Demand volume.**
*Why C.* Center-of-gravity weights coordinates by Vi (demand at each customer). It's a pure geometric centroid math.
*Why others wrong.* A (Labor) appears in factor-rating, not COG; B (Real-estate) — same; D (Tax incentives) — also factor-rating layer. COG is intentionally simple to give a baseline only.
*Takeaway.* COG = volume only; everything else comes later.

**Q3. Answer: D — Clean-slate, math-optimal site.**
*Why D.* Greenfield literally means "green field" — no existing constraint, optimize on math alone.
*Why others wrong.* A is *brownfield* (existing facilities). B (overseas) is location-agnostic to greenfield/brownfield terminology. C (LEED) is sustainability certification, unrelated.
*Takeaway.* Greenfield = blank slate; brownfield = use existing.

**Q4. Answer: A — Cross-dock.**
*Why A.* Cross-docks deliberately hold inventory for hours, not days. Throughput-focused, not storage-focused.
*Why others wrong.* B (Plant warehouse), C (DC), D (FSL) all hold inventory days-to-weeks. FSL holds smaller quantities but for similar dwell times.
*Takeaway.* Cross-dock = zero (or near-zero) dwell.

**Q5. Answer: D — Lead logistics provider, typically asset-light.**
*Why D.* 4PL = orchestrator. Does not own assets (or owns minimally). Manages other 3PLs.
*Why others wrong.* A is a 2PL (asset-based carrier). B is partial-3PL. C confuses with e-commerce platforms.
*Takeaway.* 3PL operates; 4PL orchestrates.

**Q6. Answer: A — Consolidate many small shipments.**
*Why A.* Hub-and-spoke aggregates LTL/parcel volume for the long haul (which would be uneconomical individually). FedEx Memphis, UPS Worldport are the canonical examples.
*Why others wrong.* B is point-to-point (FTL direct). C — hub-and-spoke is actually slower per shipment. D — one-to-one is direct.
*Takeaway.* Hub-and-spoke wins when consolidation economies > direct speed.

**Q7. Answer: A — √N × single-loc SS.**
*Why A.* Square-root rule of inventory consolidation/decentralization. Often abbreviated √N or √(N1/N2).
*Why others wrong.* B (N × SS) would over-state by ~3× at N=4. C (1/N) wrongly suggests inventory shrinks. D (log N) is the wrong functional form.
*Takeaway.* 4 DCs ≈ 2× SS of 1 DC; 9 DCs ≈ 3× SS of 1 DC.

**Q8. Answer: D — Holds and operates physical assets.**
*Why D.* 3PL is asset-based (warehouses, trucks). 4PL is asset-light orchestrator.
*Why others wrong.* A (cost premium) varies. B (international-only) is false. C (regulation) — both must comply with relevant regulations.
*Takeaway.* Same as Q5 — asset vs orchestration.

**Q9. Answer: C — 18.**
*Why C.* X = (4000·10 + 1000·50) / 5000 = 90,000 / 5000 = **18**. The much larger demand at A (4000) pulls the centroid toward A.
*Why others wrong.* A (30), B (22), D (40) all miscompute the weighting. Common error: just averaging the X-coordinates ((10+50)/2=30) without weighting.
*Takeaway.* Always weight by volume; bigger volumes pull harder.

**Q10. Answer: B — Defers customs duty until goods leave.**
*Why B.* FTZ defers duty until goods "enter commerce" of the host country. Can also enable duty inversion.
*Why others wrong.* A (immediate charging) is direct import. C (no storage) is wrong. D (export-only) — many FTZs allow domestic distribution after duty payment.
*Takeaway.* FTZ = deferral + (sometimes) inversion benefit.

**Q11. Answer: D — Long-term, capital-intensive, hard to reverse.**
*Why D.* Strategic = horizon 3–10 yrs, big sunk costs.
*Why others wrong.* A (daily reversal) is operational. B (identical to operational) is definitionally wrong. C (only IT) is absurd.
*Takeaway.* Time horizon + reversibility = strategic vs tactical.

**Q12. Answer: D — Hybrid (pre-build + seasonal labor + 3PL overflow).**
*Why D.* 40% in one quarter is too peaked for any pure strategy.
*Why others wrong.* A (Pure Lead) wastes capex year-round. B (Pure Lag) loses Q4 sales. C (Pure Match, fixed staff) is contradictory — match needs flex.
*Takeaway.* Heavy seasonal peaks → hybrid is the realistic answer.

**Q13. Answer: C — Theoretical × Utilization × Efficiency.**
*Why C.* Standard CLTD formula. Memorize.
*Why others wrong.* A (subtracts cushion) — cushion is 1−utilization, already in the multiplier. B (adds cushion) — wrong sign. D — service level isn't in the rated-capacity formula.
*Takeaway.* Rated = T × U × E.

**Q14. Answer: D — ~85–90%.**
*Why D.* Queueing theory (Kingman, Hopp-Spearman): wait-time → infinity as utilization → 100% with any variability. ~85% is the practical wall.
*Why others wrong.* A (25%) is wastefully low. B (50%) is below the wall. C (100% ideal) violates queueing theory.
*Takeaway.* Don't push utilization > 85% if you have variability.

**Q15. Answer: C — Σ|A−F|/N.**
*Why C.* MAD = Mean Absolute Deviation. Absolute values prevent positive/negative cancellation.
*Why others wrong.* A (no abs) is bias (signs cancel). B (squared) is MSE/variance. D (ratio) is MAPE-adjacent.
*Takeaway.* MAD has absolute values; bias doesn't.

**Q16. Answer: C — Negative bias.**
*Why C.* ASCM convention: bias = Σ(A−F)/N. If forecast over-predicts, A < F, so (A−F) is negative → negative bias.
*Why others wrong.* A (MAD-only) confuses magnitude with direction. B (positive bias) is the wrong sign under ASCM convention. D (RMSE-only) is also magnitude.
*Takeaway.* ASCM convention: over-forecast = negative bias.

**Q17. Answer: A — 4 to 6.**
*Why A.* |TS| > ~4–6 indicates the forecast model is biased and needs retuning.
*Why others wrong.* B (1) is normal. C (0.1) is unrealistically tight. D (100) is way past any failure threshold.
*Takeaway.* TS > 4 means re-tune the model.

**Q18. Answer: D — ~80% of value.**
*Why D.* Pareto principle: ~20% of SKUs ≈ 80% of value = A-items.
*Why others wrong.* A (20%) would be flat distribution. B (50%) is middle-zone. C (95%) overstates.
*Takeaway.* 80/20 rule (Pareto): A = 20% SKUs, 80% value.

**Q19. Answer: A — Identify the bottleneck.**
*Why A.* TOC Step 1: Identify. Then Exploit, Subordinate, Elevate, Repeat.
*Why others wrong.* B (Hire) is "Elevate" (Step 4). C (Buy equipment) — same, Step 4. D (Subordinate) is Step 3.
*Takeaway.* TOC ordering matters.

**Q20. Answer: B — Forecast at aggregate.**
*Why B.* Postponement delays differentiation, letting you forecast at the aggregate (less uncertainty) level.
*Why others wrong.* A (Earlier decisions) is the opposite. C (Outsourcing) is a different lever. D (Eliminate SS) overstates — SS is reduced, not eliminated.
*Takeaway.* Postponement = aggregate forecast (lower error).

**Q21. Answer: C — Cash collection.**
*Why C.* OTC = Order-to-CASH. Cash is when value is realized; DSO clock stops here.
*Why others wrong.* A (Delivery) is mid-cycle. B (Invoicing) is just before cash. D (Returns) is post-cycle if at all.
*Takeaway.* OTC ends at cash, not delivery.

**Q22. Answer: B — Net on-hand minus committed orders.**
*Why B.* ATP is *current* uncommitted stock available to promise.
*Why others wrong.* A (Net on-hand + planned receipts) is CTP, not ATP. C (Forecast) is unrelated. D (Capacity) is unrelated.
*Takeaway.* ATP = current; CTP = current + planned receipts.

**Q23. Answer: A — Includes planned receipts/production.**
*Why A.* CTP = ATP + planned inbound; lets you promise dates after a planned receipt arrives.
*Why others wrong.* B (Subtract SS) is unusual; C (Past data only) is wrong; D (Ignores allocations) — both ATP and CTP respect allocations.
*Takeaway.* CTP is future-aware.

**Q24. Answer: B — Purchase Order.**

*Why others wrong.* A (Invoice) is 810. C (ASN) is 856. D (PO Ack) is 855.
*Takeaway.* Memorize the four codes.

**Q25. Answer: D — Advance Shipping Notice.**

*Takeaway.* Same family memorization.

**Q26. Answer: C — Batch.**

**Q27. Answer: B — Outbound shipping cutoffs drive timing.**

**Q28. Answer: B — Multiplicative product.**

**Q29. Answer: D — 81.5%.**

*Takeaway.* 4 components at 95% compound to ~81.5%.

**Q38. Answer: C — 1.65.**
*Why C.* Z(95%) = 1.65. Z(99%) = 2.33.
*Why others wrong.* A (1.00) ≈ 84% service. B (1.28) is 90%. D (2.33) is 99%.
*Takeaway.* 95% = 1.65, 99% = 2.33.

**Q39. Answer: A — Amplifies variability upstream.**
*Why A.* Forrester (1958), Lee et al. (1997): demand variability gets amplified as it moves upstream.
*Why others wrong.* B (Smooths) is exactly backwards. C (Improves service) is opposite. D (Reduces LT) is unrelated.
*Takeaway.* Bullwhip = amplification.

**Q40. Answer: B — Form postponement.**
*Why B.* Benetton dyes after seeing demand → defers the form (color) until demand is known.
*Why others wrong.* A (Time) defers production. C (Place) holds at hub. D (Push) is pre-allocation.
*Takeaway.* Color/config = form postponement.

**Q41. Answer: A — Shared POS / CPFR.**
*Why A.* Information sharing damps bullwhip (Lee et al., 1997).
*Why others wrong.* B (Larger batches) worsens bullwhip. C (Lock LT) doesn't fix the cause. D (More SS) is symptom treatment.
*Takeaway.* Information sharing > inventory band-aid.

**Q42. Answer: B — 10–30%.**
*Why B.* Industry-cited MEIO benefit (CSCMP, ToolsGroup, Llamasoft case studies).
*Why others wrong.* A (1–2%) understates. C (50%) is unrealistic at scale. D (Negative) is false.
*Takeaway.* MEIO is a real lever, ~10–30%.

**Q43. Answer: B — Monthly (most frequently).**
*Why B.* A-items get the most attention; count monthly.
*Why others wrong.* A (Annually) wastes A-item priority. C (Quarterly) is B-cadence. D (Never) is absurd.
*Takeaway.* A = monthly; B = quarterly; C = annually.

**Q44. Answer: A — 95–99%.**
*Why A.* World-class DCs target this range; 100% is unrealistic.
*Why others wrong.* B (75–85%) and C (50–70%) are poor. D (Exactly 100%) is impossible at scale.
*Takeaway.* IRA target 95–99%.

**Q45. Answer: D — Reorder Point.**

*Why others wrong.* A (Receiving Ops) is invented. B (Rate Order Plan) is invented. C (Replenishment Period) is invented.
*Takeaway.* ROP = Reorder Point.

**Q46. Answer: C — New product launches and promotions.**
*Why C.* Push central allocates inventory you can't pull (it doesn't exist downstream yet).
*Why others wrong.* A (Stable staples) = pull. B (Long-tail) = pull or drop-ship. D (JIT) = pull.
*Takeaway.* Push for launches; pull for sustaining.

**Q47. Answer: A — Reviews every T periods, orders up to target.**
*Why A.* P-system = periodic review.
*Why others wrong.* B (Continuous) is Q-system. C (1-for-1) is base-stock. D (No max) — P-system *has* a max (the target).
*Takeaway.* P = periodic; Q = continuous.

**Q48. Answer: B — Collaborative Planning, Forecasting, and Replenishment.**

*Why others wrong.* A, C, D are invented phrases.
*Takeaway.* CPFR is canonical.

**Q49. Answer: C — Backward by lead time.**

*Why others wrong.* A (Forward) is opposite direction. B (Always two weeks) is wrong — it's the actual LT. D (By variability) is wrong driver.
*Takeaway.* Release = Receipt − LT.

**Q50. Answer: C — 7.2.**
